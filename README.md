## Models

| Model | Type | Notebook | Input |
|-------|------|----------|-------|
| QR1 | Quantum reservoir (1 virtual node) | `Time_serial_Finance_regression.ipynb` | 7 features |
| QR2 | Quantum reservoir (2 virtual nodes) | `Time_serial_Finance_regression.ipynb` | 7 features |
| HAR | Linear | `Reservoir_Learning.ipynb` | RV lags (1mo, 3mo, 12mo) |
| HARX | Linear + exogenous | `Reservoir_Learning.ipynb` | HAR + all macro features |
| AR(1) | Autoregressive | `Reservoir_Learning.ipynb` | 1 RV lag |
| AR(3) | Autoregressive | `Reservoir_Learning.ipynb` | 3 RV lags |
| ARMAX | AR(3) + exogenous | `Reservoir_Learning.ipynb` | AR(3) + all macro features |
| LSTM | Neural network | `LSTM.ipynb` | RV only (hidden=60) |
| LSTMX | Neural network | `LSTM.ipynb` | 11 features (hidden=50) |
| CRL | Echo state network | `classical_reservoir.ipynb` | RV only (N=50 neurons) |
| CRLX | Echo state network | `classical_reservoir.ipynb` | 11 features (N=20 neurons) |

All models use a **rolling window** of 571 months for training and predict the last 245 months (Aug 1997 – Dec 2017) out-of-sample.

## Python Pipeline

### Install dependencies

```bash
pip install numpy pandas statsmodels scikit-learn torch reservoirpy arch scipy matplotlib
```

### Run in order

```bash
# 1. Preprocess: creates Data_raw.csv and dff.csv
python preprocess.py

# 2. Train LSTM and LSTMX → writes LSTM/lstm60_predictions.csv, LSTM/lstmx50_predictions.csv
#    Open and run LSTM.ipynb

# 3. Train CRL and CRLX → writes Classical_Reservoir_learning/best_CRL(50)_predictions.csv
#    and Classical_Reservoir_learning/best_CRLX(20)_predictions.csv
#    Open and run classical_reservoir.ipynb

# 4. Run all linear models (HAR, HARX, AR, ARMAX), load all predictions, run MCS + DM tests
#    Open and run Reservoir_Learning.ipynb
```

### `preprocess.py`
Reads `Data.CSV`, runs an ADF stationarity test on each column, and first-differences any non-stationary columns (typically `DP` → `diff_DP` and `TB` → `diff_TB`). Writes `Data_raw.csv` and `dff.csv`. Must be run before any notebook.

### `LSTM.ipynb`
Implements two LSTM variants using PyTorch:
- **LSTM** — RV-only input, 2-layer stacked LSTM, `hidden_size=60`
- **LSTMX** — 11 macro-financial features, `hidden_size=50`

Both use `K=3` as sequence length, `batch_size=64`, `num_epochs=100`, `lr=0.001` (Adam).

### `classical_reservoir.ipynb`
Implements two Echo State Network (ESN) variants using `reservoirpy`:
- **CRL** — RV-only input, `N=50` reservoir neurons
- **CRLX** — 11 features, `N=20` reservoir neurons

Reservoir hyperparameters: `lr=0.6` (leak rate), `sr=0.9` (spectral radius), `input_scaling=0.1`. Output weights trained by Ridge regression (`ridge=1e-7`).

### `Reservoir_Learning.ipynb`
- Trains **HAR**, **HARX**, **AR(1)**, **AR(3)**, **ARMAX** via OLS/MLE rolling windows
- Loads predictions from all other notebooks and `predict_result.csv`
- Runs **Model Confidence Set (MCS)** test (`arch` library) with MSE and QLIKE loss
- Runs pairwise **Diebold-Mariano (DM)** tests with Newey-West HAC variance across all 11 model pairs
- Outputs `dm_stat.csv` and `p_value.csv`

## Julia Pipeline (Quantum Models) — [WIP: requires external packages]

`Time_series.jl` and `Time_serial_Finance_regression.ipynb` contain the full quantum reservoir implementation, but depend on two **custom, unpublished packages** that are not included in this repository:

- `QuantumCircuits_demo` — expected at `../package/QuantumCircuits_demo/src`
- `VQC_demo_cuda` — expected at `../package/VQC_demo_cuda/src`

An **NVIDIA GPU with CUDA support** is also required. Without these, the Julia notebooks cannot be executed.

**Pre-computed output:** `predict_result.csv` ships with the repo and contains the 245 QR1/QR2 predictions needed by `Reservoir_Learning.ipynb`. You do not need to run the Julia pipeline to reproduce the final comparison results.

### What `Time_series.jl` implements (when packages are available)

- `Qreservoir(nqubit, ps)` — builds the transverse-field Ising Hamiltonian (`H = Σ J_ij X_i X_j + Σ Z_i`)
- `Quantum_Reservoir(...)` — runs the full GPU-accelerated reservoir simulation (encode via RY gates → evolve under `e^{-iτH}` → measure `⟨Z_i⟩`)
- `Quantum_Reservoir_util(...)` — variant used for Shapley value analysis
- `Quantum_Reservoir_single(...)` — single-sample CPU version for threaded parallelism
- `MyModel` — struct storing ridge regression weights per rolling window
- Metrics: `MSE`, `RMSE`, `MAE`, `MAPE`, `compute_qlike`, `hitrate`
- Helpers: `normalization`, `denormalization`, `shift`, `rolling`, `coeff_matrix`
